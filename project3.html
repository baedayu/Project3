Project 3: Object oriented programming

[1] What is Object Oriented Programming?

1. Definition

I. Class is a data structure that has attributes and methods. Methods are procedures, functions defined within the class. Class is a kind of backbone, blueprint. We can make different objects belonging to a same class. The detailed data within attributes are different, but we can use same methods with objects.

II. Object oriented programming is one of the programming paradigms which emphasize the usefulness of objects. It makes applying others code easier. And by using functions and classes already available, the programmers can focus more on the present work.

  • Turtle: draw lines, shapes
  • Twilio: send SMS

2. Difference from using functions

I. We can import modules from Python Standard Library or external libraries to use functions. Module is a upper category consisted with functions. Function receive and process the input and return the output. There are situations when using class has more advantages.

Example) Making a movie site

  • Function: We have to use series of function to reach the final result. We have to give the entire data when calling the function. Because different movies needs different data, we should have to make separate python files for each movie. If we want to make a change within a function, we should have to change all of the separate python files.
  • Class: We make a new class needed to make a movie site. The data structure and functions are designed. Then we can make different objects of each movie to manipulate. If we want to make a change, we only have to revise the class or instance variable.

II. In large programming projects, object oriented programming is a valuable tool. We can use already available objects or make one. Then make instances and use the instance variable and method. It enables us to think and construct the blueprint first, then fill in with actual data. One by one. Separating the content and structure is similar with html and css. In the CSS file, we define the text color, font, line width ... of each class. If we change the variable within a class, the text that is classified to that class appears different automatically in the webpage.

1. Constructor

I. When we initialize a class, __init__ function is called first. This function makes space wihin the memory for the new instance. Thus the __init__ function in named the "constructor".

3. Inheritance, method overriding

I. We can reuse a part of the Class code by making a Class Child. We insert the code below inside the __init__function.

class Child(Parent):
def __init__ (self, var1, var2, var3):
Parent.__init__(self, var1, var2)

II. When we call the class Child, the var1, var2 is inherited. It equals to the class Parent's instant variable.

III. If we separately define the method within the Child class, t his overrides the Parent's definition. Otherwise it is inherited by the Parent class. Inheritance and method overriding enables both reusing the code and adding new content.
